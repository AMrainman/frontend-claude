# 测试规范

## 框架选择

从 `package.json#devDependencies` 判断，优先级：Vitest > Jest，组件测试配合 Testing Library。

## 覆盖要求

每个函数/模块至少覆盖：
- 正常路径
- 边界值（空值、最大值、类型边界）
- 异常 / 错误路径

组件测试重点：渲染正确性、用户交互行为、props 变化响应。

## 规范

- 文件命名：`xxx.test.ts` 或 `xxx.spec.ts`，放被测文件同级或 `test` 或 `__tests__/` 目录
- 描述用中文说明意图：`it('当输入为空时，应返回错误提示')`
- 禁止访问真实数据库或外部 API，必须 mock
- 禁止删除用例或写无意义断言来凑通过率

## 豁免

纯样式调整、文案修改、配置文件变更无需生成测试用例。
